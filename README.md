# TODOリストを作ろう

## 概要
個人開発初心者御用達のTODOリスト。
腐るほど世にあるので今更自作するのも死ぬほどばからしいんだけど、初心者御用達のせいで変な人の変なアプリ踏まないようにするのもだるいし
更新止まってないか確認するのもだるい。人によってほしい機能も変わるし過剰な機能はそれはそれで認知の負荷。
自分の欲しい部分だけ自作するのが無難。

## 機能
## 欲しい
- cliから開いて、どのパスから実行しても同じストレージにアクセスできる
- 日付の自動記録. (自動収集できる箇所は意識から外す)
- エディタはvim
- to think. 長くなりそうな項目をto thinkリストとして別立て。
- ライフサイクルごとの分類。寿命ごとに複数テーブル用意して、長期間未解決のものは徐々に長寿命用のテーブルに移していく
- 一覧

## いらない
- 構造化
- 通知
- 期限(きっちりとした期限あるものは別途カレンダーで管理してるので、おおまかな寿命の把握分のみ) 
- 高度な検索

## 微妙
- タグ 
  - タグベースの管理とか絶対にやりたくない
  - が、長寿命のタスクが残ってる場合タスクとして残すよりタグと言う形に変換して小タスクのみを残す+小タスクにタグを付ける方がタスク減る
  - c.f. xxxの発光過程を見る(長期間残ってるタスク)→ 1. xxxの最適化, 2. 既存実験のxxx発光スペクトル収集, 3. 最適化構造の一点計算,...に分解して 発光過程を見るタスクを消す。代わりにそれぞれのタスクにxxx発光のタグ

## 技術詳細
- uvのtoolインストールでグローバルに使えるようにする
- ホーム下にアプリ用ディレクトリ作ってそこにデータ貯める。
