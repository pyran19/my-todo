# 考え方（コンセプト）

my-todo は「自分の欲しい機能だけを備えた最小TODO」です。背景にある考え方は次の2つです。

- **認知負荷を増やさない。** 自動で記録・管理できる情報は、ユーザーの意識から外す。
- **おおまかな寿命が分かればよい。** きっちりした締切管理はしない（厳密な締切はカレンダー側で管理する想定）。

このページでは、その思想が形になっている2つの仕組みを説明します。

## ライフサイクル（寿命）による分類

タスクは寿命の長さで 3 つの段階に分類されます。

| 段階 | 意味 |
|------|------|
| `short` | 短期。追加直後はここに入る |
| `mid` | 中期。しばらく解決されずに残っているもの |
| `long` | 長期。かなり長く居座っているもの |

ポイントは、**何もしなくても古いタスクが自動的に長寿命側へ移っていく**ことです。
追加してから一定日数が経つと、`short` → `mid` → `long` と段階が上がります（既定では約 7 日で `mid`、約 30 日で `long`）。

常駐プロセスや通知は持たないため、この移行は `todo` コマンドを実行したタイミングでまとめて行われます。
これにより、一覧を見るだけで「どれくらい長く残っているタスクか」がひと目で把握できます。

`todo ls` は既定で `short` 段階のみを表示します。日々は短期タスクに集中し、長寿命側は
`todo ls -s all` などで必要なときだけ確認する、という運用を想定しています。

手動で段階を変えたいときは `todo promote` / `todo demote` を使います。

## to think リスト

「タスク」として消化するにはまだ早い、**ちょっと考える必要がある項目** を置いておく別立てのリストです。
通常のタスク一覧に混ぜず、寿命分類も完了状態も持たない、純粋な思考メモの置き場として分けています。

`todo think add` / `ls` / `edit` / `rm` で操作します。
